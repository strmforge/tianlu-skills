# Method Cycle

Tianlu treats method work as a circular generation system:

```text
absorb scoped input -> generate neutral method -> compose scoped variant -> re-enter as scoped input
```

The loop starts and ends in scoped form. The entry form is source-shaped input from a concrete event, runtime, project, tool, organization, machine, person, or task. The exit form is a new concrete variant composed from neutral methods for a specific context. Both forms are scoped by design.

Neutral generation is the reusable middle of the loop. It extracts a mechanism from scoped material, removes or contains local bias, and keeps trigger, evidence, failure mode, and boundary visible enough for later recombination.

Composition spends that neutral plasticity into a new scoped variant. This is not a loss of neutrality; it is how a reusable method becomes practical. The variant becomes risky only when it hides its scope, loses provenance, or claims universal status without neutral generation and evidence review.

The circle has three active motions:

- Absorb: carry scoped or biased source material without pretending it is already general.
- Generate: extract a neutral reusable method from that material.
- Compose: recombine neutral methods into a new scoped variant for a concrete context.

The composed variant then returns to the absorb side as fresh scoped input after practice. The system is therefore not a one-way ladder from local to general. It is a cycle where scoped practice can create neutral methods, and neutral methods can create many scoped practices.

The boundary is intentionally asymmetric: absorb receives scoped or biased material, generate produces a neutral reusable middle, and compose produces scoped or biased material again. Creation is therefore recombination, not final promotion. A composed variant starts the next turn of the cycle as scoped input, even when every ingredient was neutral.

Read the loop as scoped input, neutral generation, scoped recombination, and return. Scope is expected at the first and third positions. The safety requirement is not to remove all scoped variants, but to prevent scoped variants from skipping the neutral generation phase while claiming broad-law status.

## Phase Invariants

| Phase | Input | Output | Invariant |
| --- | --- | --- | --- |
| Absorb | Scoped or biased material | Candidate with source and scope | Do not treat source material as authority. |
| Generate | Candidate material | Neutral reusable method | Remove or contain local bias before broad reuse. |
| Compose | Neutral methods | Scoped practical variant | Name the target context and preserve provenance. |
| Re-enter | Scoped variant after use | New scoped input | Keep evidence, scope, and boundaries inspectable. |

The method system is combinatorial. Many scoped inputs can produce neutral methods, and neutral methods can be recombined into many scoped variants. Scoped variants are not pollutants; they are useful concrete forms. They simply must not pretend to be neutral law.

## Practical Rule

When adding or promoting a method, ask which phase it is in:

- If it is scoped source material, place it in incubation or an overlay.
- If it is a neutral mechanism, place it in `dao/`, `laws/`, or `index/` according to its role.
- If it is a runtime or project variant, place it in an adapter or overlay.
- If it is a variant returning from practice, record it as scoped input again before further promotion.
