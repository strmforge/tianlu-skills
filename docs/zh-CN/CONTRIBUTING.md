# 贡献说明

天箓 Skills 还在早期。任何新增内容都要保持中性、诚实标注适用域，并留下证据。

## 添加方法前先问

- 触发这个方法的场景是什么？
- 它建议的机制是什么？
- 支撑它的证据是什么？
- 已知失败模式是什么？
- 它的边界是什么？
- 它是运行时中性、adapter 专属、overlay 专属、孵化中、已废弃，还是不安全？

## 放在哪里

- 元规则放 `tiandao/`。
- 具体方法放 `laws/`。
- 路由、依赖、作用域、冲突放 `index/`。
- 运行时打包放 `adapters/`。
- 项目或个人方法放 `overlays/`。
- 未验证的新方法先放 `incubator/`。
- 验证用例放 `eval/`。

## 中性审查

不要把私有项目名、协作者代号、本机路径、运行时专属断言写进中性文件。
