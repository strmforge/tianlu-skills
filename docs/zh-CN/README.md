# 天箓 Skills 概览

天箓 Skills 是一套中性的 Agent 工作流方法系统。

它不是绑定某个模型、某个 Agent 运行时、某个项目或某种个人习惯的 skill 包。它的目标是记录方法、标注适用域、维护方法之间的关系，并用证据决定方法是否应该保留、适配、升格或废止。

## 三字模型

```text
dao   = 源
laws  = 集
index = 序
```

### 源：dao

`dao/` 放元规则：方法如何进入系统、如何确定适用域、如何成熟、如何冲突、如何升格或废止。

### 集：laws

`laws/` 放具体方法。多样性属于这里。一个方法可以很通用，也可以很窄；可以是运行时专属、项目专属、孵化中、已废弃或不安全。关键是诚实标注适用域。

### 序：index

`index/` 放秩序：路由、依赖、作用域、冲突和关系图。没有秩序的方法集合只是堆，有秩序的方法集合才可查询、可组合、可治理。

## 其他层

- `adapters/`：面向具体运行时的打包和安装说明，例如 Codex、Claude Code、Hermes。
- `overlays/`：项目、组织、机器或个人的局部方法脉络。
- `incubator/`：新方法入口，不是垃圾桶。
- `eval/`：验法题和期望行为。

## 什么适合进入这里

适合：

- 能写清 trigger、mechanism、evidence、failure mode、boundary 的方法。
- 可以诚实标注适用域的 Agent 工作流经验。
- 未来可能被不同运行时 adapter 使用的通用法。
- 虽然不通用但足够有价值、适合留在 overlay 的局部方法。

不适合：

- 没有证据的一次性观点。
- 放错位置的运行时专属说明。
- 出现在中性文件里的私有项目术语。
- 绕过 review 的安装指令。

## 当前种子法

- Agent 编排：任务边界、委派、角色、共享状态。
- 浏览器与运行时：浏览器委托权限、运行时回执、远程控制。
- CI 与 DevOps 工作流：输入来源、事件内容污染、修复通道、扫描器证据、端点库存、策略即代码。
- 证据与验法：来源证据、replay、独立验证。
- 安装与供应链：不可信安装文本、持久化、依赖安装出口。
- MCP、工具与授权：权限边界、最小 allowlist、绑定授权、运行时输出处理。
- 记忆与学习：上下文工作集、持久记忆写入、生命周期、轨迹、信息流学习、skill 生成。
