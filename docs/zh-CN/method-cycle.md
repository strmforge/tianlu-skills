# 方法圆环

天箓把方法工作看成一个圆形生成系统：

```text
吸收有偏输入 -> 生成中立方法 -> 组合有偏变体 -> 回流为有偏输入
```

这个圆从有作用域的形态开始，也回到有作用域的形态。入口处是来自具体事件、运行时、项目、工具、组织、机器、个人或任务的材料。出口处是由中立方法重新组合出的具体变体。两者天然都有偏向。

中立生成是圆里的可复用中间形态。它从有偏材料中抽出机制，移除或容纳局部偏向，并让 trigger、evidence、failure mode、boundary 保持可见，方便之后再次组合。

组合会把中立的可塑性用到新的具体场景里，生成新的有偏变体。这不是中立性的失败，而是中立方法落地的方式。危险不在有偏，而在隐藏作用域、丢掉来源，或没有经过中立生成和证据审查就宣称自己是普遍方法。

这个圆有三个主动动作：

- 承：承载有作用域、有偏向的资讯或方法，不把来源材料直接当作普遍方法。
- 生：从候选材料里抽出中立、可复用的方法。
- 创：把中立方法重新组合成面向具体运行时、项目、组织、机器或任务的有偏版本。

创出的有偏版本经过实践后，会重新回到承的位置，成为下一轮有偏输入。因此这不是一条只从局部升到通用的单向梯子，而是一个圆：有偏实践可以生出中立方法，中立方法也可以创出许多新的有偏实践。

相位边界必须保持清楚：承接住的是有作用域或有偏材料，生得到的是中立可复用的中间形态，创再把这个中间形态重新组合成有作用域或有偏材料。创不是最终升格；哪怕所有材料都来自中立方法，创出的版本也要作为新的有偏输入回到承的位置。

## 相位约束

| 相位 | 输入 | 输出 | 不变量 |
| --- | --- | --- | --- |
| 吸收 | 有作用域或有偏材料 | 带来源和作用域的候选 | 不把来源材料直接当权威。 |
| 生成 | 候选材料 | 中立可复用方法 | 广泛复用前先移除或容纳局部偏向。 |
| 组合 | 中立方法 | 有作用域的实用变体 | 标清目标场景，并保留来源。 |
| 回流 | 使用后的有偏变体 | 新的有偏输入 | 证据、作用域和边界仍可检查。 |

这个方法系统是可组合的。许多有偏输入可以生成中立方法，中立方法又可以重新组合成许多新的有偏变体。有偏变体不是污染物，而是有用的具体形态；它只是不该冒充中立法。

## 实用规则

添加或升格方法时，先判断它处在哪个相位：

- 如果它是有作用域的来源材料，放入 incubator 或 overlay。
- 如果它是中立机制，按角色放入 `dao/`、`laws/` 或 `index/`。
- 如果它是运行时或项目变体，放入 adapter 或 overlay。
- 如果它是实践后回流的变体，先重新记录为有偏输入，再谈进一步升格。
