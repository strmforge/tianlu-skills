# Incubator

The incubator is the entrance for new methods.

A method in incubation is not rejected. It is waiting for scope, evidence, and placement.

## Incubation Template

```text
Title:
Signal:
Source:
Proposed trigger:
Proposed mechanism:
Initial scope:
Known failure modes:
Verification needed:
Placement candidates:
```

## Current Incubating Methods

- [Agent team meta-factory review](agent-team-meta-factory-review.md): review tools that generate agent teams, skills, orchestrators, runtime pointers, or future behavior files before activation.
- [AI company benchmark readiness review](ai-company-benchmark-readiness-review.md): separate AI CEO, AI employee, one-person company OS, digital-worker benchmark, and agentic-market claims from business execution authority before promotion or adoption.
- [DLP alert triage evidence review](dlp-alert-triage-evidence-review.md): shape Microsoft Purview DLP alert, incident, matched-condition, triage-agent, and remediation-candidate signals into evidence packets and owner handoffs without authorizing tenant access, policy mutation, remediation execution, legal/compliance conclusions, or sensitive-data retention.
- [Developer extension marketplace readiness review](developer-extension-marketplace-readiness-review.md): shape Chrome Web Store, VS Code Marketplace, extension manifest, permissions, privacy, listing, publisher, package, review, and publication signals into readiness packets without authorizing install, package execution, account setup, credential use, upload, publication, unpublish, privacy/legal/security conclusions, or revenue proof.
- [Generative media production planning](generative-media-production-planning.md): shape AI video, avatar, anime, UGC, and generated-media requests into production briefs, source ledgers, run receipts, QA passes, and publication handoffs without granting generation, upload, publish, rights, or account authority.
- [Install-time package firewall](install-time-package-firewall.md): package-manager-path enforcement for agent package installs; not active protection and not a tool endorsement.
- [Instruction-file policy profile](instruction-file-policy-profile.md): scoped persistent instruction profiles for safer command lanes, file handling, and skill discovery.
- [Local service missed-call intake review](local-service-missed-call-intake-review.md): shape missed-call, AI receptionist, call tracking, and text-back signals into intake evidence packets and owner handoffs without authorizing calls, SMS, account configuration, CRM writes, bookings, compliance conclusions, or customer contact.
- [Local review response evidence review](local-review-response-evidence-review.md): shape local reviews, testimonials, reputation-management, and AI review-response signals into public source maps, official-policy tables, unsent draft queues, and owner handoffs without authorizing public replies, review requests, incentives, reports, suppression, account actions, customer contact, or compliance conclusions.
- [Local agent provider gateway](local-agent-provider-gateway.md): review model-routing shims, provider dashboards, config patchers, vision bridges, and desktop/MCP control surfaces before adoption.
- [MCP API bridge adoption review](mcp-api-bridge-adoption-review.md): review official, hosted, compressed, Code Mode, portal-backed, registry-published, or API-wrapper MCP bridges before connection, install, OAuth, token use, or tool registration.
- [Open-domain industry horizon scout](open-domain-industry-horizon-scout.md): maintain bounded proactive industry/source-map coverage for recurring feed-reading workflows without treating trends as authority or waiting for user-named topics.
- [Open source sponsorship readiness review](open-source-sponsorship-readiness-review.md): shape GitHub Sponsors, repository `FUNDING.yml`, sponsor button, external funding platform, fiscal-host, Open Collective, tier, payout, invoice, receipt, expense, and tax-information signals into sponsorship readiness packets without authorizing repository writes, payment setup, sponsor contact, legal/tax/accounting conclusions, or income proof.
- [Public skill collection quality gate](public-skill-collection-quality-gate.md): sample and score public skill packs as method catalogs before importing, installing, or composing local variants.
- [RFP response compliance matrix review](rfp-response-compliance-matrix-review.md): shape seller-side RFP, proposal, questionnaire, and compliance-matrix signals into source inventories, claim-evidence ledgers, amendment trackers, reviewer queues, and no-submit handoffs.
- [Solo revenue claim verification review](solo-revenue-claim-verification-review.md): verify solo-founder, one-person-company, MRR, ARR, revenue, no-employees, or one-laptop business claims before treating them as cases or opportunity evidence.
- [Source-grounded structured data readiness review](source-grounded-structured-data-readiness-review.md): shape messy documents, websites, and AI-search visibility signals into source inventories, schemas, cited extraction QA, and owner handoffs without claiming tool accuracy, ranking, revenue, production readiness, or account-write authority.
- [Software supply chain evidence triage service](software-supply-chain-evidence-triage-service.md): shape SBOM, VEX, Dependabot, advisory, KEV, OpenSSF, and SLSA signals into reviewable evidence and handoff packets without installing, patching, releasing, or claiming compliance.
- [Terminal output compression review](terminal-output-compression-review.md): review local wrappers, hooks, and summaries that compress noisy command output while preserving raw evidence.
- [Voice profile skill review](voice-profile-skill-review.md): review corpus-to-profile-to-skill systems for consent, privacy, measured evidence, held-out verification, activation, misuse, and rollback boundaries.
