# Playbook Registry

The playbook registry lists output-oriented methods and what each one helps an agent produce.

Playbooks are not permission grants. They are productive method cards. When a playbook touches account actions, money, publication, regulated claims, private data, or durable behavior, route to the supporting law family before acting.

## Registry Rules

- Add productive methods here only after they have a method card in `playbooks/`.
- Name the useful output, not only the safety boundary.
- Keep platform-specific tricks, account-specific preferences, and project voice in `overlays/` or adapters.
- Use `laws/` only when a playbook crosses an authority, evidence, privacy, install, or durable-write boundary.
