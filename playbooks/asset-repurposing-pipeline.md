# Playbook: Asset Repurposing Pipeline

Status: seed draft

Scope: domain-neutral. This playbook helps an agent transform one source asset into multiple scoped variants while preserving the core promise and constraints. It is not tied to one social platform, marketplace, document format, brand voice, genre, or media tool.

## Purpose

Help an agent produce a multi-format asset package: long-form draft, short post, script, product-page section, email, support answer, slide outline, or internal brief variant, with a mapping table and consistency checks.

This playbook also covers source-material-to-content-package requests when the useful output is a reviewable bundle of variants. If the source material first needs evidence synthesis, start with `research-to-brief-synthesis.md`; if the request is mainly about calendars, distribution, or feedback loops, compose with `content-operations.md`.

## Inputs

- Source asset such as a brief, article, transcript, product note, story scene, customer answer, research summary, or campaign idea.
- Target formats and audiences.
- Source-material context such as transcript, interview, webinar, notes, PDF, report, meeting material, or research summary when the task is to create a content package.
- Core promise, claims, tone, style, and required omissions.
- Channel constraints such as length, structure, media type, accessibility, claim policy, or review requirements.
- Existing brand, project, or author guidance.
- Success metric such as clarity, reuse speed, consistency, conversion, reader engagement, or stakeholder approval.

## Procedure

1. Extract the core asset units.
   - Identify the main promise, audience, proof points, narrative beats, examples, objections, and calls to action.
   - Mark claims that need evidence or domain review.

2. Define target variants.
   - Name each format, audience, purpose, constraints, and review owner.
   - Keep platform-specific mechanics as scoped variant notes, not neutral method rules.

3. Map units to variants.
   - Decide which source units each variant keeps, compresses, expands, reframes, or omits.
   - Record why a unit changed so later reviewers can audit the transformation.

4. Draft the variants.
   - Preserve the core promise while adapting structure, pacing, length, and emphasis.
   - Avoid mechanical copying when the target format needs a different reading or viewing experience.

5. Run consistency checks.
   - Check factual claims, numbers, names, dates, links, continuity, tone, and forbidden claims.
   - Confirm that each variant still serves its own audience and purpose.

6. Prepare the review packet.
   - Provide the variants, mapping table, open questions, claim status, and publication or handoff checklist.
   - Separate draft variants from live publication, upload, send, or account mutation.

## Outputs

- Multi-format asset package.
- Source-material content package when requested.
- Source-to-variant mapping table.
- Variant-specific notes.
- Claim and consistency checklist.
- Review packet with open questions and next actions.

## Quality Signals

- Each variant preserves the source asset's core promise but fits its target format.
- The mapping table explains major changes instead of hiding them.
- Factual claims and regulated claims are marked for evidence or review.
- The output avoids one-size-fits-all reposting.
- Reviewers can approve, reject, or revise each variant independently.

## Human Checkpoints

- Brand voice, author voice, creative direction, or public stance.
- Publication, upload, email send, message reply, ad launch, or account action.
- Copyrighted material, private data, user-generated content, or living-person likeness.
- Legal, medical, financial, safety, political, or other regulated claims.

## Evidence

Source signal: cross-domain method review identified asset repurposing as a shared productive need for commerce, content, writing, support, and analysis agents.

Validation state: seed draft. Needs replay on real cross-format content, commerce, writing, and documentation tasks before promotion.

Known failures:

- Reposting the same text into every channel without adapting the audience experience.
- Losing the original claim, proof, or story logic during compression.
- Creating variants that conflict with each other.
- Publishing or sending variants without human review where the channel or domain requires it.
